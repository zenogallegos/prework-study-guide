HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • We can use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • To link CSS to HTML, we use the element "link" in the "head" section of HTML with rel="stylsheet" and href="[stylesheet location]"

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Typing 'git add -A' will commit all files to the working branch
  • 'git commit -m' will commit the changes to the repository
  • Always use 'git pull origin main' before pushing to pull the base branch into your current branch
  • Use 'git push origin (name of branch)' to push your commit
  • Update the local repository by using 'git checkout main' and then 'git pull origin main'

JavaScript

  • To link JavaScript to HTML, we use the element "script" within the "body" element of the HTML document with src="[Javascript location]
  • An array can hold various types of data that all use one variable. This is done by using var variable-name=["name", "name", "name"]
  • Arrays: each item in an array is assigned a number that starts from 0 and goes up a whole number for each item within the array. An array might be constructed like this:

    var shapes = ["triangle", "square", "pentagon", "circle"];

  • For Statements: these statemetns can list items in an array by using a loop. a loop can be created from an array as follows:

    var shapes = ["triangle", "square", "pentagon", "circle"];

    for (var x = 0; x < shapes.length; x++) {

       console.log(shapes[x]);

    }

    The "var x = 0" defines the starting point and identifies which index of the array to target first, "x < shapes.length" gives a boolean comparison, which is essential a condition for how many times the loop will repeat, where "shapes" is the variable of the entire array and "length" is the amount of variables within the array, where the loop will repeat as long as the length of the array is more than x, and "x++", shorthand for "x + 1" is what will occur every time the loop runs. so in short, a loop will go like this:

    for ([starting point], [sets limits for the loop], [what happens for every loop])
